How the Tribeca Task Tool works

A plain-English instruction is understood by AI, confirmed by a person, turned into a real Google Calendar event and email, and the assignee's reply is captured back against the task. The flow below shows each node and exactly what lives inside it.

1   Browser — Frontendhosted on GitHub Pages

Single-page app (docs/index.html)

Screens: New Task, Confirm & Edit, Dashboard (search / filters / autocomplete), Detail (reply thread). Sends the public anon key on every request. Talks only to Supabase.

▼2  User types an instruction → POST /parse  ·  Dashboard loads via GET /tasks
3   Supabase — back officedatabase + serverless functions

Edge Functions (Deno · verify_jwt = false)

parseapprovetasksinboundapp

OpenAI — called inside /parse

Model: gpt-4o-mini, structured outputs (JSON schema, strict). Key read from tt_config, never leaves the server. First name then matched against tt_contacts for a real email.

Postgres tables

tt_tasks — canonical task record (owns the task_id, status, calendar link, latest reply)  ·  tt_contacts — known name → real full name + email
tt_config — private settings: openai_api_key, n8n_outbound_webhook_url  ·  tt_replies — full reply thread (one row per reply)

▼4  On approve → POST /approve saves the task, then sends the payload to the n8n webhook (URL from tt_config)
5   n8n — automationcloud (two workflows)

Outbound workflow

Webhook → Is calendar_event? → Create Calendar Event (adds assignee as attendee → invite) → Send Email (subject tag [Tribeca Task #id]) → Respond (returns the calendar link).

Inbound workflow

Gmail Trigger (polls ~1 min) → Extract + Clean (reads the #id tag, strips quoted history) → POST /inbound (with the anon apikey header).

▼6  n8n creates the event & sends the email through Google
7   Google — Calendar + Gmailvia n8n's authorized account

What the assignee receives

A Google Calendar invitation (Accept / Maybe / Decline) and a formatted email with the event details and reply instructions.

Reply loop — The assignee replies to the email → it lands in the shared Gmail inbox → the n8n Inbound workflow picks it up within ~1 minute → POST /inbound → the reply is saved to tt_replies and the task status becomes Reply received → it appears on the Dashboard.

Architecture & End-to-End Flow

Step-by-step

The same flow, described one step at a time, with the node each step runs in.

  1. Open & type. The user opens the app and types a plain-English instruction. Frontend · GitHub Pages
  2. Understand. The app calls POST /parse, which sends the text to OpenAI gpt-4o-mini with a strict JSON schema to extract the task fields. Supabase /parse → OpenAI
  3. Resolve the person. Still inside /parse, the first name is looked up in tt_contacts; a match replaces the assignee name and email with the real one. Supabase · tt_contacts
  4. Confirm. The structured fields fill the Confirm & Edit card; the user reviews, edits if needed, and clicks Approve. Frontend
  5. Save. POST /approve generates the task_id, writes the row to tt_tasks, and reads the n8n webhook URL from tt_config. Supabase /approve
  6. Hand off to automation. /approve sends the task payload to the n8n Outbound webhook. Supabase → n8n
  7. Create & notify. n8n branches on task type, creates the Google Calendar event (assignee added as attendee, so Google emails an invite), and sends the email tagged [Tribeca Task #id]; the returned calendar link is stored on tt_tasks. n8n → Google
  8. Reply. The assignee replies to the email; it arrives in the shared Gmail inbox. Google Gmail
  9. Capture. The Inbound Gmail Trigger polls (~1 min), Extract + Clean reads the #id and the reply text, and posts to POST /inbound with the anon apikey. n8n → Supabase /inbound
  10. Store. /inbound appends the reply to tt_replies and updates tt_tasks (status = Reply received). Supabase · tt_replies
  11. View. The Dashboard (GET /tasks) shows the task and its full reply thread. Frontend ← Supabase

Where each thing lives

Frontend (GitHub Pages)

  • Single-page app: docs/index.html
  • Screens: New Task, Confirm, Dashboard, Detail
  • Sends the public anon key; calls Supabase only

Supabase (back office)

  • Functions: parse, approve, tasks, inbound, app
  • AI: OpenAI gpt-4o-mini (inside /parse)
  • Tables: tt_tasks, tt_contacts, tt_config, tt_replies

n8n (automation)

  • Outbound: Webhook → Calendar + Email → Respond
  • Inbound: Gmail Trigger → Extract → POST /inbound
  • Holds the authorized Google sign-in

Google

  • Calendar: the event + RSVP invite
  • Gmail: sends the email, receives the reply
  • Reply returns through n8n Inbound
